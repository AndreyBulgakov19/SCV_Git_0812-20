# Работа с Git

## 1. Установка Git и Visual Studio Code (Семинар 1). 
Загружаем последнии версии ПО: 

Git 

`https://git-scm.com/downloads` 

Visual Studio Code 

`https://code.visualstudio.com`

Устанавливаем продукты с настройками по умолчанию.

## 2. Проверка наличия установленного Git.

Запускаем Visual Studio Code, открываем окно терминала `Вид/Терминал`.
В терминале выполнить команду `git version`

Если Git установлен, появится сообщение с инф. о версии программы. Иначе будет сообщение об ошибке.

## 3. Настройка Git.
При первом использовании необходимо представится.
Для этого в терминале надо выполнить 2 команды.
```
git config --global user.name «Ваше имя английскими буквами»
git config --global user.email ваша почта@example.com
```

## 4. Инициализация репозитория Git.

После установки и настройки Git, необходимо создать пустую папку на локальном диске, например: `GITINSTRUCTION`. Запустить `Visual Studio Code` и открыть в нем указанную папку выбрав команду `Файл / Открыть папку...`.  Для начала отслеживания изменений Git указанной папки необходимо нициализировать в ней репозиторий, командой:

`git init`

## 5. Добавление файлов в репозиторий.
Сначало необходимо создать файл в папке репозитория, используя команду Visual Studio Code `Файл/Создать файл...` (например Git_Instraction.md), после чего, для добавления этого файла в репозиторий, выполнить команду в терминале  `git add` с указанием имени файла. Например:

`git add Git_Instraction.md`

*ps. расшерение .md - формат Markdown*

## 6. Проверка статуса репозитория

Просмотреть статус  репозитория и определить, какие файлы в каком состоянии находятся можно командой:

`git status`

## 7. Запись изменений в репозиторий.

После того как в файл внесены изменения, необходимо выполнить команду: `git add` с указанием имени файла. Например:

`git add Git_Instraction.md`

После, выполнить команду `git commit` с опцией -m "message",  в кавычках указать описание (message) к данному коммиту, например:

`git commit -m "message"`

В некоторых ситуациях можно совместить команды `add, commit` применив в команде `git comit` опцию `-am`, например:

`git commit -am "message"`

в этом случае, исполнять команду `git add` не требуется.

Внести изменения в message последнего коммита можно с помощью опции --amend. Например:

`git commit --amend -m "Updated message"`

## 8a. Просмотр изменений в Git.

Чтоб просмотреть изменения в файле по сравнению с последним коммитом, введите команду:
`git diff`

## 9a. Просмотр истории изменений в Git.

Просматривать изменения, внесённые в репозиторий, можно с помощью команды 

`git log`

Чтоб вывести информацию в сокращенном виде введите:

`git log --oneline`

## 10a. Переключение между ветками в Git.

Переключатся между ветками можно с помощью команды `git checkout` в которой необходимо указать номер ветки (или первых 4 символа), который можно получить командой `git log`.
Например:

`git checkout bd93`

Для того, чтоб переключиться на ветку `"master"`, необходимо ввести команду:

`git checkout master`

Для того, чтобы исключить из отслеживания в репозитории определенные файлы или папки, необходимо содать файл ***.gitignore*** и записать в него их названия или шаблоны, соотв. таким файлам или папкам.

## 9. Создание веток в Git.
Ветка в Git - это простой перемещаемый указатель из коммитов, обычно последний в цепочке еоммитов.
По умолчанию имя основной ветки в Git - *master*/

Создать ветку можно командой: 

```
git branch <имя новой ветки>
```

В результате создается новый указатель на текущий коммит.

Список веток в репозитории можно посмотреть с помощью команды:

```
git branch
```
Текущая ветка будет отмечана звездочкой (*): **\*master**

## 10. Слияние веток и разрешение конфликтов.

Для слияния выбранной ветки с текущей нужно выполнить команду:

```
git merge <Название выбранной ветки>
```

Если была изменена одна и таже часть файла, то может возникнуть конфликт, который потребует участия пользователя. VSCode предлагает варианты разрешения.
